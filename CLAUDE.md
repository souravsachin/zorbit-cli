# Zorbit Service: zorbit-cli

## Purpose

This repository implements the **zorbit-cli** tool for the Zorbit platform.

Zorbit is a MACH-compliant shared platform infrastructure used to build enterprise applications.

The CLI provides scaffolding and management commands for Zorbit platform development, including creating services, registering events and privileges, and generating APIs.

## Responsibilities

- Scaffold new NestJS services following Zorbit architecture patterns
- Create modules within existing services
- Register events in the platform event registry
- Register privileges in the platform privilege registry
- Generate full CRUD APIs following Zorbit REST grammar
- Enforce Zorbit conventions (hash IDs, namespace isolation, event-driven integration)

## Architecture Context

This service follows Zorbit platform architecture.

Key rules:

- REST API grammar
- namespace-based multi-tenancy
- short hash identifiers
- event-driven integration
- service isolation

## Dependencies

Allowed dependencies:

- zorbit-core (for registry files and templates)
- commander (CLI framework)
- inquirer (interactive prompts)
- chalk (terminal output styling)
- handlebars (template rendering)
- fs-extra (file system utilities)

Forbidden dependencies:

- direct database access to other services
- cross-service code imports

## Platform Dependencies

Upstream services:
- zorbit-core (event-registry, privilege-registry, service-templates)

Downstream consumers:
- All Zorbit service repositories (generated by this CLI)


## Commands

| Command | Description |
|---------|-------------|
| `zorbit create-service <name>` | Scaffold a new NestJS service |
| `zorbit create-module <name>` | Create a module within an existing service |
| `zorbit register-event <domain.entity.action>` | Register an event in the platform registry |
| `zorbit register-privilege <code>` | Register a privilege in the platform registry |
| `zorbit generate-api <resource>` | Generate a full CRUD API for a resource |

## Development Guidelines

Follow Zorbit architecture rules.
All generated code must follow Zorbit conventions for REST grammar, namespace isolation, hash IDs, and event-driven integration.
